A Comprehensive Guide to Sorting Algorithms

1Ô∏è‚É£ Selection Sort

üîëConcept: Find the minimum element in the unsorted part of the array and swap it with the element at the beginning of the unsorted part. Repeat this process until the entire array is sorted.

üíª Code

void selectionSort(int arr[], int n){
    for(int i = 0; i < n - 1; i++){
        int min_ind = i;
        for(int j = i + 1; j < n; j++){
            if(arr[j] < arr[min_ind]) min_ind = j;
        }
        swap(arr[min_ind], arr[i]);
    }
}

üîÑ Dry Run (arr = [29, 10, 14, 37])

  • Pass 1: min=10 ‚Üí [10, 29, 14, 37]
  • Pass 2: min=14 ‚Üí [10, 14, 29, 37]
  • Pass 3: min=29 ‚Üí [10, 14, 29, 37]

üÉè Flashcards

Q: Time complexity of Selection Sort?

A: O(n¬≤) (always, even if sorted).

Q: Is Selection Sort stable?

A: ‚ùå No.

2Ô∏è‚É£ Quick Sort (Hoare Partition)

üîëConcept: Choose a 'pivot' element. Partition the array such that all elements smaller than the pivot come before it, and all elements greater come after. Recursively apply this to the sub-arrays.

üíª Code

int partition(int a[], int low, int high){
    int pivot = a[low], i = low - 1, j = high + 1;
    while(true){
        do{ i++; }while(a[i] < pivot);
        do{ j--; }while(a[j] > pivot);
        if(i >= j) return j;
        swap(a[i], a[j]);
    }
}

void quickSort(int a[], int low, int high){
    if(low < high){
        int p = partition(a, low, high);
        quickSort(a, low, p);
        quickSort(a, p + 1, high);
    }
}

üÉè Flashcards

Q: Average time complexity of Quick Sort?

A: O(n log n).

Q: Worst-case?

A: O(n¬≤) (when the pivot is always the smallest or largest element).

Q: Is Quick Sort stable?

A: ‚ùå No.

3Ô∏è‚É£ Merge Sort

üîëConcept: A "divide and conquer" algorithm. It divides the array into two halves, recursively sorts them, and then merges the two sorted halves back together.

üíª Code

void merge(int arr[], int l, int m, int r){
    int n1 = m - l + 1, n2 = r - m;
    int L[n1], R[n2];
    for(int i = 0; i < n1; i++) L[i] = arr[l + i];
    for(int j = 0; j < n2; j++) R[j] = arr[m + 1 + j];
    int i = 0, j = 0, k = l;
    while(i < n1 && j < n2){
        if(L[i] <= R[j]) arr[k++] = L[i++];
        else arr[k++] = R[j++];
    }
    while(i < n1) arr[k++] = L[i++];
    while(j < n2) arr[k++] = R[j++];
}

void mergeSort(int arr[], int l, int r){
    if(l < r){
        int m = l + (r - l) / 2; // Avoids overflow for large l, r
        mergeSort(arr, l, m);
        mergeSort(arr, m + 1, r);
        merge(arr, l, m, r);
    }
}

üÉè Flashcards

Q: Time complexity of Merge Sort?

A: O(n log n) (always).

Q: Space complexity?

A: O(n).

Q: Is Merge Sort stable?

A: ‚úÖ Yes.

4Ô∏è‚É£ Insertion Sort

üîëConcept: It builds the final sorted array one item at a time. It iterates through the input elements and inserts each element into its correct position in the sorted part of the array.

üíª Code

void insertionSort(int arr[], int n){
    for(int i = 1; i < n; i++){
        int key = arr[i];
        int j = i - 1;
        while(j >= 0 && arr[j] > key){
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}

üîÑ Dry Run (arr = [5, 3, 4])

  • i=1, key=3: Compare 3 with 5. Shift 5. Insert 3 ‚Üí [3, 5, 4]
  • i=2, key=4: Compare 4 with 5. Shift 5. Insert 4 ‚Üí [3, 4, 5]

üÉè Flashcards

Q: Best case complexity?

A: O(n) (when the array is already sorted).

Q: Worst case?

A: O(n¬≤).

Q: Is Insertion Sort stable?

A: ‚úÖ Yes.

5Ô∏è‚É£ Bubble Sort

üîëConcept: Repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order. The pass through the list is repeated until the list is sorted. The largest element "bubbles" to the top.

üíª Code

void bubbleSort(int arr[], int n){
    for(int i = 0; i < n - 1; i++){
        bool swapped = false;
        for(int j = 0; j < n - 1 - i; j++){
            if(arr[j] > arr[j + 1]){
                swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }
        if(!swapped) break; // Optimization for already sorted array
    }
}

üîÑ Dry Run (arr = [5, 1, 4, 2])

  • Pass 1: [1, 5, 4, 2] ‚Üí [1, 4, 5, 2] ‚Üí [1, 4, 2, 5]
  • Pass 2: [1, 4, 2, 5] ‚Üí [1, 2, 4, 5]
  • Pass 3: No swaps. Array is sorted.

üÉè Flashcards

Q: Time complexity (worst)?

A: O(n¬≤).

Q: Best case (already sorted with optimization)?

A: O(n).

Q: Is Bubble Sort stable?

A: ‚úÖ Yes.

6Ô∏è‚É£ Heap Sort

üîëConcept: It's a comparison-based sorting algorithm that uses a binary heap data structure. It first converts the array into a max heap. Then, it repeatedly swaps the root (maximum element) with the last element, reduces the heap size, and calls heapify on the root to maintain the heap property.

üíª Code

void heapify(int arr[], int n, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;

    if (left < n && arr[left] > arr[largest])
        largest = left;

    if (right < n && arr[right] > arr[largest])
        largest = right;

    if (largest != i) {
        swap(arr[i], arr[largest]);
        heapify(arr, n, largest);
    }
}

void heapSort(int arr[], int n) {
    // Build max heap
    for (int i = n / 2 - 1; i >= 0; i--)
        heapify(arr, n, i);

    // Extract elements from heap
    for (int i = n - 1; i > 0; i--) {
        swap(arr[0], arr[i]);
        heapify(arr, i, 0);
    }
}

üÉè Flashcards

Q: Time complexity of Heap Sort?

A: O(n log n) (for all cases).

Q: Space complexity?

A: O(1) (in-place).

Q: Is Heap Sort stable?

A: ‚ùå No.
